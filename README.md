# Aspiorhynchus laticeps comparative genomics

Reproducible analysis code for the comparative-genomics study of the endangered
plateau cyprinid **_Aspiorhynchus laticeps_** and **_Diptychus maculatus_** within
a 12-species cypriniform phylogenomic framework. This repository is the code
package referenced in the paper's Code Availability statement.

The analysis asks whether two ecologically similar but phylogenetically distinct
plateau fishes follow a shared genomic trajectory, examined at four levels:
gene-repertoire composition, gene-family size change, coding-sequence divergence,
and chromosome-scale organization.

> **This repository contains code and environment specifications only.** No genomic
> data or bulk intermediates are stored here. Every input and intermediate is
> either downloadable from a public source (NCBI, eggNOG, KEGG) or regenerated by
> the notebooks — see **[DATA.md](DATA.md)**.

## Environment setup

The supported, reproducible setup is via **Conda**. Create the environments you
need from `env/*.yml`:

```bash
conda env create -f env/env_biopython.yml          # core Python (Biopython, MAFFT, IQ-TREE, ...)
conda env create -f env/env_orthofinder.yml        # OrthoFinder v3 + DIAMOND
conda env create -f env/env_cafe5.yml              # CAFE5 gene-family dynamics
conda env create -f env/env_paml.yml               # PAML (MCMCTree, yn00/codeml)
conda env create -f env/eggnog_env.yml             # eggNOG-mapper
conda env create -f env/bioconductor-clusterprofiler_env.yml   # R enrichment (clusterProfiler, ggtree)
```

`env/sum.list` lists the key tool versions. Each notebook states which environment
it runs under.

> **Docker note.** `.build/Dockerfile` references an internal image
> (`huang/comparative-genomics:20250828`) that is **not** published to a public
> registry, so `docker-compose` cannot be built as-is outside the authors'
> environment. It is retained only to document the original container. Reproduce
> the analysis with the Conda environments above.

## Data acquisition (quick start)

```bash
# 1. Download the 12 genome assemblies from NCBI (requires ncbi-datasets-cli)
bash scripts/download_genomes.sh

# 2. (Optional) download the eggNOG database if re-running annotation from scratch
conda run -n eggnog_env download_eggnog_data.py -y --data_dir DB/Eggnog
```

See **[DATA.md](DATA.md)** for the full dependency map. Reproducing from raw
genomes (RepeatMasker → BRAKER3 → OrthoFinder) is compute-intensive; the
downstream analyses (Stages 2–8) run quickly once orthogroups and annotations
exist.

## Pipeline

Notebooks are numbered by stage. Run them in order within the matching Conda
environment (e.g. `quarto render Notebooks/<name>.qmd`).

| Stage | Notebook(s) | Purpose → key outputs |
|---|---|---|
| 0 | `0-ultimate-env_construction`, `0-ultimate-preprocessing` | Tool setup; RepeatMasker soft-masking + BRAKER3 annotation |
| 1 | `1-ultimate-OrthoFinder_preparation-step{1,2}` | Extract & standardize proteomes (longest isoform, header normalization) |
| 2 | `2-ultimate-AL_OrthoFinder` | OrthoFinder v3 orthogroups, species tree, gene duplications |
| 2 | `2-ultimate-PhylogeneticTreeEstimation` | Supermatrix → IQ-TREE → MCMCTree dated tree (`scripts/build_supermatrix_tree.py`, `setup_mcmctree.py`) |
| 2 | `2-ultimate-CAFE-analysis` | CAFE5 gene-family expansion/contraction |
| 2 | `2-ultimate-KaKs-analysis`, `2-ultimate-Genome_wide_selection_screen` | Ka/Ks (MAFFT → PAL2NAL → yn00); genome-wide selection screen |
| 3 | `3-ultimate-Visualization4OrthoFinder` | Orthogroup overview, Jaccard heatmap, UpSet, trees |
| 4 | `4-ultimate-EnrichmentAnalysis-preparing` | eggNOG annotation → build `org.*.eg.db` |
| 4 | `4-ultimate-EnrichmentAnalysis4SpecificOGs`, `...-KaKs` | GO/KEGG enrichment on CAFE and Ka/Ks categories |
| 5 | `5-ultimate-PastML` | Ancestral-state reconstruction (supplementary) |
| 6 | `6-ultimate-MCScanX` | MCScanX synteny + corrected 4DTv (`scripts/run_mcscanx_pairwise.py`, `run_4dtv_analysis.py`) |
| 7–8 | `7-ultimate-StateOfTheArt-Visualization`, `8-ultimate-GOKO-Visualization` | pyCirclize circos; cross-category (CAFE × Ka/Ks) enrichment figures |

`docs/genome-preprocessing-guide.qmd` is a standalone how-to for the Stage 0
masking + annotation steps.

## Two-set design (focal-species switching)

The same 12-species OrthoFinder run supports two focal-species analyses, selected
by a YAML config:

- **Set 2** — *Aspiorhynchus laticeps* focal (`Notebooks/config/paths-set2.yaml`) — **committed default**
- **Set 3** — *Diptychus maculatus* focal (`Notebooks/config/paths-set3.yaml`)

To switch, change the `config_path` at the top of a notebook (or set
`GENOMICS_CONFIG_PATH`). No other code changes are needed.
