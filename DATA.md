# Data acquisition & regeneration

This repository ships **code and environment specifications only** — no genomic
data or bulk intermediates are committed. Every input and intermediate used by
the analysis is either **downloadable from a public source** or **regenerated by
the notebooks/scripts** in this repository. This document maps each dependency to
how you obtain it.

All paths below are given relative to the repository root, which is mounted at
`/home/jovyan` inside the analysis container.

## Dependency map

| Dependency | Size | How to obtain | Produced/used by |
|---|---|---|---|
| **Raw genome assemblies** (12 species) | ~30 GB | `bash scripts/download_genomes.sh` (NCBI Datasets CLI, reads `data/accessions.tsv`) | input to preprocessing |

| **Repeat-masked genomes** | — | RepeatModeler + RepeatMasker | `Notebooks/0-ultimate-preprocessing.qmd`, `scripts/run_repeatmasker_*.sh` |
| **Gene annotations** (`braker.aa` / `braker.codingseq` / `braker.gff3`) | — | BRAKER3 on the soft-masked genomes | `Notebooks/0-ultimate-preprocessing.qmd` |
| **Proteomes** (`*.faa`, longest isoform) | ~0.4 GB | Extracted from BRAKER output | `Notebooks/1-ultimate-OrthoFinder_preparation-step{1,2}.qmd` |
| **OrthoFinder results** (`Results_Mar03/`) | ~5 GB | OrthoFinder v3 (DIAMOND backend) on the 12 proteomes | `Notebooks/2-ultimate-AL_OrthoFinder.qmd` |

## Starting point

Reproducing the full pipeline from raw genomes (RepeatMasker → BRAKER3 →
OrthoFinder) is compute-intensive. Stages 0–1 (preprocessing and orthology
inference) are the expensive, one-time upstream steps; the **downstream analyses
(Stages 2–8)** — gene-family dynamics, selection, enrichment, synteny — are the
scientifically interesting parts and run quickly once the OrthoFinder results and
eggNOG annotations exist.
